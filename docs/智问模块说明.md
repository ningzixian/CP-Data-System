# 智问模块使用说明

## 一、这是什么

**智问** 是 CP-Data-System 新增的**自然语言查询模块**。用户用中文问一句话，模块自动解析、查询、可视化回答。

三层智能：
1. **规则引擎**（毫秒级）—— 匹配常见查询模式
2. **LLM 兜底**（秒级，本地）—— 规则搞不定的复杂问题交给本地大模型
3. **智能提示** —— 根据当前数据状态动态生成"猜您想问"

- 数据源：现成的 CP 检测数据（mock）+ `frontend/public/data/` 下的管网 GIS 数据
- 实现：浏览器内执行，**零后端依赖**，**零外网调用**
- 部署：内网服务器上 build 完扔 nginx 即可

## 二、怎么跑

### 开发模式（带热更新）
双击根目录的 `start.cmd`，浏览器打开 `http://localhost:5173`

### 生产预览
双击根目录的 `start-prod.cmd`，先 build 再用 vite preview 启动，端口 8080

### 手动命令
```powershell
cd frontend
node_modules\.bin\vite.cmd          # 开发
node_modules\.bin\vite.cmd build    # 构建
node_modules\.bin\vite.cmd preview  # 预览
```

## 三、查询流程

```
用户问题
   ↓
规则引擎评估置信度
   ├─ ≥ 0.4 → 规则解析（毫秒）
   └─ < 0.4 → LLM 兜底
                ├─ LLM 可用 → 调本地 LLM 解析 → 引擎执行
                └─ LLM 不可用 → fallback 提示
```

置信度评分依据：
- 命中数据集关键词（管线/调压箱/检测等）+0.3
- 命中小区（七里/三里/六里）+0.1
- 命中过滤（压力/材质/管径/状态）+0.1~0.15
- 命中聚合（多少/总长/平均/最大/分布）+0.2~0.3

## 四、能问什么

### 规则引擎能直接答的（毫秒级）
- 数量类：七里调压箱多少个 / 三里管线有多少
- 统计类：七里DN100以上管线总长 / 管地电位平均值
- 分布类：管线压力分布 / 各小区管径分布
- 过滤类：三里低压管线 / 七里钢管 / DN200以上
- 检测类：异常检测记录 / 进度异常的小区
- 地图类：六里调压箱 / 三里引入口

### LLM 兜底能答的（秒级，需开启本地 LLM）
- 自由问句：「这周最值得关注的异常是什么？」
- 复合查询：「七里和六里谁的 DN100 管线更多？」
- 模糊描述：「管道老化情况怎么样？」
- 业务推理：「防腐层检测异常说明了什么？」

LLM 拿到问题后输出结构化 JSON 查询计划，再用真实数据算出结果，**避免了大模型瞎编数字**。

## 五、LLM 兜底配置

### 1. 启动 Ollama
```bash
# 安装 Ollama（https://ollama.com）
ollama serve                    # 启动服务（默认 11434 端口）
ollama pull qwen2.5:7b          # 拉模型（推荐 7B，CPU/GPU 都能跑）
```

### 2. 在智问页面配置
点击右上角 ⚙ 齿轮按钮：
- **启用 LLM**：✅
- **API 风格**：Ollama
- **Base URL**：`http://localhost:11434`
- **模型**：`qwen2.5:7b`
- **Temperature**：0.1（要稳定输出）
- 点击「测试连接」验证

### 3. 也支持 OpenAI 兼容 API
vLLM / LocalAI / LM Studio / 一键本地 GPT 都能用：
- **API 风格**：OpenAI 兼容
- **Base URL**：`http://localhost:1234`（LM Studio 默认）
- **API Key**：按你的服务要求填
- **模型**：按实际加载的填

配置存浏览器 localStorage（key `zhiwen.llm`），刷新页面不丢。

## 六、智能提示（猜您想问）

智问页面初始欢迎区会展示 **6 张动态提示卡片**，按数据状态实时生成：

| 类型 | 触发条件 | 例子 |
|---|---|---|
| ⚠️ 异常 | 有 N 条异常检测记录 | 「有 3 条异常检测记录，涉及防腐层检测等」 |
| 📋 待办 | ≥ 5 条待开始记录 | 「20 条检测待开始」 |
| 🐢 进度慢 | 有 < 50% 进度的单元 | 「三里、六里的腐控单元进度较慢」 |
| 📏 长度对比 | 多个小区有管线 | 「七里管网最长，总长 8.5 km」 |
| 🔩 材质分布 | 管网有数据 | 「看看管线材质分布」 |
| 🏭 调压箱 | 调压箱数据 | 「各小区调压箱数量」 |

点击卡片直接发起查询。

数据变化时自动刷新。

## 八、支持的过滤维度

- **小区**：七里 / 三里 / 六里（可省略表示全部）
- **管径**：DN100、100mm，可加「以上/以下/大于/小于」
- **压力等级**：低压 / 中压A / 中压B
- **材质**：钢管 / PE管
- **检测状态**：异常 / 待开始 / 进行中 / 通过

## 九、回答格式

每次查询返回 4~5 部分：

1. **自然语言回答**（顶部，带高亮关键词 + 规则/LLM 标签）
2. **详细数据表**（el-table，最多 50 条）
3. **可视化图表**（饼图 / 柱状图）
4. **地图高亮**（右侧 AMap 地图自动适配视野，不同小区用不同颜色）
5. **结构化解释**（"我把您的问题理解成了..."）

LLM 兜底时还会展示 LLM 的原始 JSON 输出，方便调试。

## 十、添加新区/新设施

1. 把新小区的 5 类 CSV 放到 `frontend/public/data/`
2. 在 `frontend/src/zhiwen/dataLoader.ts` 的 `COMMUNITIES` 数组加上小区全名
3. 重启 dev server 即可
4. 智能提示会自动包含新区

## 十一、导出数据

每次查询后，**答案卡片右上角**会有「📥 导出」按钮。两步引导：

1. **是否导出** — 弹窗提示当前命中数和已展示条数，按"导出 / 取消"
2. **以什么格式** — 选择 Excel / CSV / JSON，可选列、行数范围

### 三种格式对比

| 格式 | 优点 | 适用场景 |
|---|---|---|
| **Excel (.xlsx)** | 双 Sheet（数据 + 元信息），列宽自适应，UTF-8 | 报告、打印、共享给非技术人员 |
| **CSV (.csv)** | 体积最小，UTF-8 BOM（Excel 打开不乱码）| 大数据量、导入其他系统、脚本处理 |
| **JSON (.json)** | 含完整元信息（问题、SQL、摘要）| 程序化处理、API 集成、备份 |

### 文件名自动生成
格式：`智问_<问题摘要>_<时间戳>.<ext>`

例如：`智问_七里DN100以上管线总长_20260717_143052.xlsx`

### Excel 文件结构
- **Sheet 1「查询结果」**：明细数据，列宽自适应
- **Sheet 2「查询元信息」**：查询问题、SQL 思路、回答摘要、命中数等

### 依赖
- 已装 `xlsx ^0.18.5`（动态 import，不影响主包体积）
